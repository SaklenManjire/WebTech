PROJECT OVERVIEW
================
"This is a Student Expense Tracker application that helps students 
manage their daily expenses, categorize their spending, and analyze 
their spending patterns. It's built with React and JavaScript, 
demonstrating clean code practices and modern web development."

KEY FEATURES TO HIGHLIGHT
==========================

1. CORE FUNCTIONALITY
   - Add new expenses (category, amount, description, date)
   - View all expenses in a list format
   - Delete expenses when needed
   - Filter expenses by category
   - Persistent data storage using localStorage

2. STATISTICS & ANALYTICS
   - Total spending calculation
   - Spending breakdown by category
   - Visual progress bars showing proportional spending
   - Real-time updates as expenses are added/deleted

ARCHITECTURE EXPLANATION
========================

1. COMPONENT-BASED DESIGN
   - ExpenseForm.jsx: Handles user input
   - ExpenseList.jsx: Displays all expenses
   - ExpenseItem.jsx: Individual expense card
   - CategoryFilter.jsx: Filter buttons
   - Statistics.jsx: Spending breakdown visualization
   
   → Benefits: Reusable, maintainable, testable

2. CUSTOM HOOK (useExpenses.js)
   - Centralizes all business logic
   - Manages state with useState
   - Syncs with localStorage using useEffect
   - Provides clean interface: addExpense(), deleteExpense(), etc.
   
   → Benefits: Separation of concerns, easy to test

3. CLEAN JAVASCRIPT
   - No TypeScript, just clean JavaScript
   - Meaningful variable and function names
   - Inline comments explain complex logic
   
   → Benefits: Simple, readable, easy to understand

ANSWERS TO COMMON VIVA QUESTIONS
==================================

Q: Why did you choose React?
A: React uses functional components and hooks for state management,
   making the code clean and maintainable. It's also industry-standard
   and perfect for building interactive UIs.

Q: Why JavaScript instead of TypeScript?
A: For a college project, JavaScript is simpler and more straightforward.
   The code is clean and readable with meaningful names and comments,
   so type safety wasn't necessary. It also reduces build complexity.

Q: How does state management work?
A: We use React's useState for expense list and selected category.
   useEffect hook syncs with localStorage for persistence. All logic
   is in a custom hook (useExpenses) for clean separation.

Q: Why localStorage?
A: It provides simple, browser-based persistence. Perfect for a 
   student project - no backend needed, data survives page refresh.

Q: How do you handle form validation?
A: In ExpenseForm.jsx, we check:
   - Amount is not empty and > 0
   - Description is not empty
   - Show alert if validation fails

Q: How are expenses filtered?
A: When user clicks a category button, we update selectedCategory state.
   The filteredExpenses array is calculated by filtering expenses array
   based on selectedCategory (or showing all if selectedCategory = 'All').

Q: How do you calculate statistics?
A: In useExpenses hook, we use reduce() to sum amounts per category,
   then calculate percentage for progress bars. Re-calculated whenever
   expenses change.

Q: Why separate components into smaller files?
A: Single Responsibility Principle - each component does one thing.
   Makes code easier to read, test, and reuse.

Q: How does data persistence work?
A: We use localStorage.setItem() to save data whenever expenses change,
   and localStorage.getItem() to load data when app starts. This happens
   automatically through useEffect.

Q: Can you extend this application?
A: Yes! Easy extensions would be:
   - Add budget limit feature
   - Export data as CSV
   - Add monthly/weekly reports
   - Charts for visualization
   - Search functionality

DATA FLOW EXPLANATION
====================

User Action → Component State Update → localStorage Save → 
Recalculate Totals → Re-render Components → Display Updated UI

Example Flow for Adding Expense:
1. User fills form and clicks "Add Expense"
2. ExpenseForm validates input
3. Calls onSubmit callback (addExpense)
4. useExpenses.addExpense() creates new Expense with unique ID
5. Updates expenses state
6. useEffect detects state change
7. Saves to localStorage
8. Components re-render with new data
9. Statistics automatically recalculate
